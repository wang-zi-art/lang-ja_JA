:tips_tweaks_general_plug:
> 一般的な推奨事項：
>
> + 静的 IP アドレスの使用は、特定の場合に役立ちます。NFS 共有フォルダは IP アドレスで参照されます。ルーターで IP アドレスを予約するか、競合を防ぐために非常に高い IP アドレスを設定してください。
> + ジャンボフレームの使用は推奨されません。ジャンボフレームは一部の Docker で問題を引き起こすことが知られています。
> + Unraid サーバーが UPS に接続されていない限り、'vm.dirty_ratio' を 20% 以上に設定しないでください。停電時にディスクに書き込まれていないデータが失われる可能性があります。
> + 'vm.dirty_background_ratio' の値を 'vm.dirty_ratio' の 1/2 以上に設定することは、両方の値に一桁の数値を使用する場合を除き、推奨されません。
:end

:tips_tweaks_nic_flow_control_plug:
> NIC フロー制御を無効にするには 'Yes' に設定します。NIC フロー制御は元々、異なる速度のデバイスが混在する
> ネットワークを支援するために実装されました。今日のネットワークではもはや必要ないようです。マネージド
> スイッチをお持ちの場合、フロー制御は通常スイッチ上でオン/オフを切り替えることができます。コンシューマー
> グレードのスイッチをお持ちの場合、そのレベルの制御はできません。フロー制御は特定の状況でネットワーク
> 問題を引き起こすことが知られています。ネットワーク問題が発生している場合は、フロー制御を無効にすることをお勧めします。
>
> 任意の設定から 'Default' に変更するには再起動が必要です。
:end

:tips_tweaks_nic_offload_plug:

:tips_tweaks_dirty_background_plug:
> 推奨値は：'vm.dirty_background_ratio' を 3%、'vm.dirty_ratio' を 5% に設定します。
> 大量の RAM がある場合は、'vm.dirty_background_ratio' を 1%、'vm.dirty_ratio' を 2% に設定します。
> 注意：Unraid を確実にシャットダウンできる UPS がない限り、これらの値をデフォルト設定以上に増やさないでください。

> 'vm.dirty_background_ratio' を総 RAM のパーセンテージとして設定します。
> 大量の RAM がない限り、デフォルト設定の 3% が推奨されます。メディアストリーミングまたはゲーム用 VM を実行する場合は、1% の値を使用してください。
:end

:tips_tweaks_cpu_governor_plug:
> 使用する通常の CPU スケーリングガバナーを選択します。
>
> + **Power Save** - 最大限の省電力のために CPU を最低周波数にロックします。
> + **On Demand** - ACPI CPUFreq ドライバのデフォルトです。負荷に基づいて CPU 周波数をスケーリングします。周波数変更は積極的です。
> + **Conservative** - 負荷に基づいて CPU 周波数をスケーリングします。より穏やかな周波数変更。レイテンシが発生する可能性があります。
> + **Performance** - 負荷に関係なく CPU を最高周波数にロックします。パフォーマンスとレイテンシを改善できます。
> + **Schedutil** - Linux スケジューラを使用して負荷に基づいて CPU 周波数をスケーリングします。
>
> 'rc.tweaks' スクリプトで CPU ガバナーを設定できるようになりました。次のコマンドを使用します：
> /usr/local/sbin/rc.tweaks set_governor 'governor'
> 'governor' の選択肢は：'ondemand'、'performance'、'powersave'、'conservative'、および 'schedutil' です。
:end

:tips_tweaks_power_save_cpu_governor_plug:
> 使用する省電力 CPU スケーリングガバナーを選択します。

> + **Power Save** - 最大限の省電力のために CPU を最低周波数にロックします。
> + **On Demand** - ACPI CPUFreq ドライバのデフォルトです。負荷に基づいて CPU 周波数をスケーリングします。周波数変更は積極的です。
> + **Conservative** - 負荷に基づいて CPU 周波数をスケーリングします。より穏やかな周波数変更。レイテンシが発生する可能性があります。
> + **Schedutil** - Linux スケジューラを使用して負荷に基づいて CPU 周波数をスケーリングします。
:end
